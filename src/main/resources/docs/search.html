Search

The search window can be activated by pressing CMD/CTRL f, or via the menu bar. After entering a few characters, search results start to appear below. The search results are organized as a keyword-in-context (KWIC) list: The found string is shown in the center (and bold), accompanied by left and right context.

Individual findings can be selected with mouse and keyboard. If one or more finding is selected, the buttons on the left become active. They allow three operations:

  • Add to existing entity: Adds all selected findings (without their context) to the entity that is selected in the tree of the main window.
  • Add to new entity: Creates a new entity and adds all selected findings to it.
  • Clear: Removes all findings

Regular Expressions

The search interface can be used with regular expressions, making it very powerful. Regular expressions allow the definition of patterns involving some variability. They can become very complex, as the full range of Java-based regular expressions can be used.

As a quick reference, the following table contains the most important bits:

SymbolDescriptionExample ExpressionExample Findings
.Matches any single symbold.g"dog", "deg", "d g", ...
[ ]Matches any of the characters in brackets alternativelyb[au]nk"bank", "bunk"
[^ ]Matches none of the characters in bracketsa[^bc]*d"ad", "aefghd", "axd"
?Makes the preceding character optionals?he"she", "he"
*Matches the preceding character 0 or more timesdog*"do", "dog", "dogg", "doggg", ...

Example Expressions

The following list shows a few examples that are useful for annotation purposes:

  • \b[^a]+\b: Find words that do not contain the letter "a"
  • \bdogs?\b: Find singular and plural forms of the word "dog"
  • \b[A-Z][a-z]*\b: Find words starting with upper case
